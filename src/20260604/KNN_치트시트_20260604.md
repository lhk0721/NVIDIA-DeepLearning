## 1. 머신러닝 전체 흐름 (가장 중요한 뼈대)

머신러닝이든 딥러닝이든 **아래 7단계**를 따른다.

| 단계 | 내용 | 사용 도구 | 왜 필요한가 |
|------|------|-----------|-------------|
| 1 | 학습시킬 **dataset 선택** | - | 문제 정의가 없으면 어떤 특성/타깃을 쓸지 정할 수 없어서 |
| 2 | **전처리** (훈련용 특성데이터 선택) | python, numpy, pandas | 모델은 정제된 수치 배열만 먹기 때문에 (결측·스케일·형태 정리) |
| 3 | 학습/평가/예측 데이터 **분리** | sklearn | 학습에 쓴 데이터로 평가하면 점수가 부풀려져 일반화 성능을 못 봄 |
| 4 | **모델 준비** (knn, random forest, svm …) | sklearn | 문제 성격에 맞는 알고리즘을 골라야 성능이 나와서 |
| 5 | train 데이터로 **학습** → `fit` | sklearn | 모델이 데이터의 패턴을 갖게 하는 단계 |
| 6 | **성능평가** → `score` (기준 미달 시 1번으로 loop) | sklearn | 실제 쓸 만한지 객관적 수치로 확인해야 해서 |
| 7 | 새로운 데이터에 대해 **예측** → `predict` | sklearn | 학습의 최종 목적이 미지의 데이터를 맞히는 것이라서 |

- **1~2단계**까지는 python · numpy · pandas 개념이 필요.
- **3단계 이후**부터 sklearn(사이킷런) 라이브러리를 활용한다.

## 2. KNN (K-Nearest Neighbor) 핵심 개념

### 한눈에 보기

| 항목 | KNN이 하는 일 | 왜 그런가 |
|------|----------------|-----------|
| **k** | 학습 산출물 ❌ → 사람이 정하는 **하이퍼파라미터** | 데이터로부터 학습되는 값이 아니라 "이웃 몇 개를 볼지" 설계 선택이라서 |
| **fit (학습)** | 훈련 데이터(특성+타깃)를 **저장만** 함 | KNN엔 추정할 가중치·계수가 없어서 — 데이터 자체가 모델 |
| **predict (예측)** | 거리 계산 → k개 이웃 → **다수결** (실제 계산은 여기서) | 저장해둔 점들과의 거리를 그때그때 재야 가까운 이웃을 알 수 있어서 |
| **학습 vs 예측** | 동일하지 않음. 오히려 **예측 때 다 함** | fit은 저장뿐, 실제 연산은 predict에 몰려 있어서 (게으른 학습) |

### 특징 (게으른 학습, Lazy Learning)
- 분류와 회귀 **모두** 사용 가능한 알고리즘. ← 이웃의 다수결이면 분류, 이웃의 평균이면 회귀라서
- **학습 = 데이터 저장**. 산재된 특성데이터를 메모리에 올려버린다. ← 따로 학습할 파라미터가 없어서
- 단점:
  - **고사양 메모리** 필요 ← 예측 때 비교하려면 훈련 데이터를 전부 들고 있어야 해서
  - **연산자원** 많이 필요, **속도 느림** ← 예측마다 모든 점과의 거리를 다시 계산해서
  - 최적의 **k 선택이 어려움** ← k가 작으면 과적합, 크면 과소적합 + 데이터 의존성·클래스 불균형·평가 변동성까지 얽혀서

### 왜 지도학습인가?
- 핵심은 예측이 아니라 **훈련을 보는 것**.
- 예측할 특성데이터에 대응하는 **타깃(정답)이 존재**하기 때문에 지도학습. ← 정답을 보고 짝지어 학습하므로

## 3. 특성 데이터(x) vs 타깃 데이터(y)

| 용어 | 변수 예 | 의미 | 생선 예시 |
|------|---------|------|-----------|
| **특성 데이터** (feature, x) | `fish_data` | 입력. "문제지"의 단서들 | 생선의 길이 + 무게 |
| **타깃 데이터** (target, y) | `fish_target` | 정답. 맞혀야 할 답 | 도미(1) / 빙어(0) |

> **`fit(x, y)` = `fit(특성, 타깃)` = `fit(문제, 정답)`** → 이 둘을 짝지어 외워두는(저장) 것이 KNN의 학습.

### 데이터 모양 규칙
- **한 행** `[25.4, 242.]` = 샘플 한 개(생선 한 마리)
- **한 열** = 하나의 특성 (1열=길이, 2열=무게)
- **순서가 중요**: `fish_data`의 0번 샘플이 도미면 `fish_target`의 0번도 반드시 1이어야 한다. ← x[i]와 y[i]를 인덱스로 짝지어 학습하므로, 순서가 어긋나면 엉뚱한 정답을 학습함

## 4. 주요 메서드 정리

### 학습 메서드
```python
knnmodel.fit(x, y)   # x: 특성데이터, y: 타깃데이터 (저장)
```

### 평가 메서드
```python
knnmodel.score(x, y)  # predict + accuracy_score 연결하여 정확도 계산
```

### 예측 메서드
```python
knnmodel.predict(x_new)  # 특성데이터만 전달! 2차원 배열 기대
```
- ⚠️ **타깃 데이터는 주면 안 된다** ← 타깃을 맞히는 게 목적이므로 입력에 정답을 넣으면 의미가 없음.
- ⚠️ **2차원 배열**로 줘야 한다. 한 개 샘플이어도 `[[ ]]`. ← predict는 "샘플들의 묶음(행=샘플, 열=특성)"을 받도록 설계돼서

### 이웃 조회 메서드
```python
distance, indexes = knnModel.kneighbors([[25,150]])
# 인자 기준 거리가 가까운 k개 데이터의 (거리, 인덱스) 반환
```

## 6. ⚠️ 시행착오 & 함정 모음

### (1) `predict`는 항상 2차원 배열
> **왜**: 입력을 "행=샘플 × 열=특성" 행렬로 받기 때문. `[30,420]`은 1차원이라 "특성 2개짜리 샘플 1개"인지 "샘플 2개"인지 모호 → 에러.
```python
knnmodel.predict([30, 420])     # ❌ 오류! shape 안 맞음
knnmodel.predict([[30, 420]])   # ✅ [1]  (한 마리여도 [[ ]])
knnmodel.predict([[30,420],[9,12]])  # ✅ [1 0]
```

### (2) target(y)은 1차원 배열을 요구
> **왜**: 타깃은 "샘플당 정답 하나"라서 형태가 `(n,)`이어야 함. DataFrame 한 열은 2차원처럼 잡혀 형태 경고/오류가 날 수 있음.
- 데이터프레임으로 접근 시 → **Series**로 주자. (한 열만 뽑으면 1차원 Series)
- 그래도 오류 나면 → **`.values`** 로 순수 ndarray 값을 주자. (pandas 인덱스 메타데이터를 떼어내서)

### (3) 넘파이는 수치 인덱싱만 지원 (팬시 인덱싱)
- 넘파이엔 암묵적 인덱스만 있어 **수치로만 접근**.
- 연속이면 슬라이싱, **떨어져 있으면 fancy indexing** 사용.
- ⚠️ **튜플로 주면 안 됨** → 서로 다른 축의 인덱스로 해석. **리스트로** 주자.
```python
arr = np.arange(5,19).reshape(7,2)
index = [1,3,5]          # 리스트! (1,3,5) 튜플 ❌
xdata = arr[index, 0]    # [7 11 15]
ydata = arr[index, 1]    # [8 12 16]
```
- `(index, 0)` 처럼 더 큰 인덱싱 튜플의 **한 자리**에 리스트가 들어가면 array-like로 취급됨. (최상위 튜플은 "축 구분 문법", 대괄호 안은 항상 객체 하나)

### (4) 우분투에서는 그래프를 반드시 save
> **왜**: 서버/WSL 등 GUI 디스플레이가 없는 환경에선 `plt.show()` 창이 안 뜨므로, 파일로 저장해야 결과를 볼 수 있음.
```python
plt.savefig('fishdata.jpeg')   # plt.show() 대신 저장해야 확인 가능
```

### (5) 정규화 안 하면 잘못된 예측 + 음수 좌표
- `train_data_split.py`에서 (25,150) 예측 시 → 정규화 안 돼 잘못된 이웃 선택. ← 무게(0~1000)가 길이(10~41)보다 스케일이 커서 거리를 독점하기 때문 (자세히는 9장)
- 오버플로우 방지 차원에서도 정규화는 필수. ← 값이 큰 상태로 곱 연산이 누적되면 수치가 폭주(overflow)할 수 있어서

### (6) 넘파이 산술연산은 position별 연산
- `dot`(행렬곱)을 제외한 산술연산은 넘파이에서 **위치별(element-wise) 연산** → **shape를 맞춰줘야** 한다. ← 같은 위치끼리 더하고 곱하므로, 형태가 다르면 짝지을 수 없음(broadcasting 규칙에 안 맞으면 에러)

## 7. 배열 합치기 — 함수별 차이 (헷갈림 주의)

| 함수 | 역할 | 입력 형태 | 결과 | 왜 튜플로 묶나 |
|------|------|-----------|------|----------------|
| `np.column_stack((a, b))` | 1차원 배열들을 **열로 쌓기** | 튜플로 묶어 전달 | 2차원 (열 방향) | 인자가 "쌓을 배열들의 묶음" 하나라서 |
| `np.concatenate((a, b))` | 두 배열 **병합** | 튜플로 **하나**만 받음 | 이어붙임 | 인자를 1개만 받도록 설계돼 여러 배열을 하나로 pack해야 해서 |
| 리스트 내포 + `np.array` | 짝지어 2차원화 | `[[l,w] for l,w in zip(...)]` | 2차원 | 각 샘플을 `[길이,무게]` 행으로 만들어야 모델 입력 형태가 돼서 |

```python
# column_stack: 두 배열을 튜플로 묶어 열로 쌓기
fish_data = np.column_stack( (length, weight) )
# [[ 25.4 242. ]
#  [ 26.3 290. ] ...]

# concatenate: 튜플로 pack해서 하나로! (concatenate는 인자 하나만 받음)
fish_target = np.concatenate( (np.ones(35), np.zeros(14)) )

# 리스트 내포 문법으로 2차원 만들기
fish_data = np.array([[l,w] for l,w in zip(length,weight)])
```

> 💡 `np.column_stack((iris_x, iris_y))` 처럼 특성+타깃을 이어붙여 DataFrame 만들 때도 사용.
> ⚠️ pandas의 concat과는 다르다.

## 8. 학습/평가 데이터 분할 — `train_test_split`

```python
from sklearn.model_selection import train_test_split

(train_x, test_x, train_y, test_y) = train_test_split(
    fish_data,
    fish_target,
    stratify=fish_target,   # 클래스 비율 유지하며 분할
    random_state=42         # 재현성 (shuffle 시드)
)
```
- 데이터셋을 **학습셋과 테스트셋으로 나눠야** 한다. ← 학습에 안 쓴 데이터로 평가해야 진짜 일반화 성능을 알 수 있어서
- 데이터 순서도 **random하게 shuffle** 해야 한다. ← 원본이 도미35개+빙어14개처럼 정렬돼 있으면, 안 섞고 자르면 한 클래스만 train/test에 쏠림
- **반환 순서 주의**: `train_x, test_x, train_y, test_y` (x들 먼저, y들 나중). ← 순서를 잘못 받으면 특성을 타깃 자리에 넣는 치명적 버그
- `stratify=y`: 분할 후에도 클래스 **비율을 유지**. ← 불균형 데이터에서 한쪽 클래스가 test에 거의 안 들어가는 걸 막아서
- `random_state=42`: 같은 시드 → 같은 분할. ← shuffle을 고정해 실험 결과를 **재현**할 수 있게 하려고

## 9. ⭐ 표준점수 정규화 (StandardScaler) — 가장 중요한 교훈

### 왜 필요한가? (정규화 하나로 예측이 뒤집힌다)

같은 새 생선 (길이 25, 무게 150) 예측 결과:

| | 5개 이웃 | KNN 예측 |
|--|----------|----------|
| **정규화 전** | 도미 1 + 빙어 4 | 빙어 ❌ (틀림) |
| **정규화 후** | 도미 5 | 도미 ✅ (맞음) |

### 원인: 유클리드 거리는 모든 축에 같은 자(ruler)를 댄다
```
거리² = (길이차)² + (무게차)²
```
- 두 특성의 스케일 차이가 약 **32배**:
  - 길이 범위 ≈ 31 (10~41)
  - 무게 범위 ≈ 993 (7~1000)
- 가장 가까운 이웃의 거리 분해:
  - `length 기여 = 0.2 (0.0%)`
  - `weight 기여 = 8464.0 (100.0%)` ← **무게가 거리를 100% 지배**
- 즉, 정규화 전 KNN은 사실상 **"무게만 보는 1차원 분류기"**. 길이는 무시됨.

### 그림의 착시
- 원래 스케일 그래프는 y축(무게) 0~1000, x축(길이) 10~41 → 세로축이 눌려 보임.
- 눈엔 도미 근처로 보여도, **알고리즘이 보는 거리 공간은 찌그러진 공간**.
- 정규화 후엔 두 축이 똑같이 −1.5~1.7 → 공간이 안 찌그러져 모든 특성이 공평하게 "투표".

### 한 줄 정리
> **표준점수 정규화 `(x - mean) / std`** 는 각 특성을 "평균에서 표준편차 몇 칸 떨어졌나"라는 **공통 단위**로 바꿔준다. 그래야 거리 기반 모델(**KNN, K-means, SVM** 등)에서 스케일 큰 특성 하나가 거리를 독점하는 걸 막고, 모든 특성이 공평하게 반영된다.

### 구현 (직접 계산)
```python
mean = np.mean(train_x, 0)   # 열(축0)별 평균 → [27.30, 454.10]
std  = np.std(train_x, 0)    # 열별 표준편차 → [9.98, 323.30]

# 표준점수 = (특성 - 평균) / 표준편차   ※ broadcasting
trained_scaled = (train_x - mean) / std
```

### ⚠️ 핵심 함정: 학습·예측·평가 모두 **같은 mean/std**로 정규화
```python
# 새 데이터 예측 시에도 train의 mean/std 사용!
newData     = ([25, 150] - mean) / std
test_scaled = (test_x   - mean) / std   # 평가도 동일

knnmodel.fit(trained_scaled, train_y)
knnmodel.score(test_scaled, test_y)     # acc: 1.0
```
- ⚠️ test나 새 데이터를 자기 자신의 mean/std로 정규화하면 **scale이 안 맞아** 틀린다.
- 산점도 모양은 정규화 전후가 **같다** (scale만 다를 뿐).

## 11. 모델 평가 지표 — 혼동행렬 & 분류 리포트

```python
from sklearn.metrics import confusion_matrix      # 혼동행렬 생성
from sklearn.metrics import classification_report # 정확도·정밀도 등 출력
```

### 이진 분류 혼동행렬
```python
y_true = [0, 1, 1, 1, 0, 1, 0]
y_pred = [0, 1, 1, 0, 1, 1, 1]

confusion_matrix(y_true, y_pred)
# [[1 2]      행=실제, 열=예측, 0부터 시작
#  [1 3]]
```

### 다중 분류 혼동행렬
```python
y_true = [0,1,1,1,0,1,0,2,2,3,3,3]
y_pred = [0,1,1,0,1,1,1,2,1,3,3,2]

confusion_matrix(y_true, y_pred)
# [[1 2 0 0]
#  [1 3 0 0]
#  [0 1 1 0]
#  [0 0 1 2]]
```

> **혼동행렬을 왜 보나**: accuracy 한 숫자로는 "어떤 클래스를 무엇으로 헷갈리는지"가 안 보여서. 대각선=정답, 비대각선=오답 위치로 약점을 짚을 수 있음.

### classification_report 주요 지표
| 지표 | 의미 | 왜/언제 보나 |
|------|------|--------------|
| **precision** (정밀도) | 예측이 양성이라 한 것 중 실제 양성 비율 | 오탐(거짓양성)의 비용이 클 때 — 스팸 오분류처럼 |
| **recall** (재현율) | 실제 양성 중 양성으로 맞춘 비율 | 놓치면 안 될 때 — 암 진단처럼 누락(거짓음성)이 치명적일 때 |
| **f1-score** | precision·recall의 조화평균 | 둘 사이 trade-off를 한 수치로 보고 싶을 때 |
| **support** | 각 클래스의 실제 샘플 수 | 클래스 불균형 여부 파악 + 평균 가중치 근거 |
| **accuracy** | 전체 정확도 | 빠른 전체 감 — 단, 불균형 데이터에선 오해 소지 |
| **macro avg** | 클래스별 단순 평균 | 소수 클래스도 똑같이 중요하게 볼 때 |
| **weighted avg** | support 가중 평균 | 데이터 비율을 반영한 현실적 평균이 필요할 때 |

## 12. 핵심 한 줄 요약 (빠른 참조)

1. 머신러닝은 **dataset → 전처리 → 분할 → 모델 → fit → score → predict** 7단계.
2. KNN은 **게으른 학습**: fit=저장, predict=거리계산+다수결. k는 하이퍼파라미터(default 5).
3. `predict`는 **항상 2차원 배열**, target(y)은 **1차원**(필요시 `.values`).
4. 넘파이 fancy indexing은 **리스트**로 (튜플 ❌, 축 구분으로 해석됨).
5. `column_stack`은 튜플로 열쌓기, `concatenate`는 튜플 하나로 병합.
6. `train_test_split` → `(train_x, test_x, train_y, test_y)`, `stratify`+`random_state`.
7. **거리 기반 모델(KNN)은 정규화 필수**: `(x-mean)/std`. 학습·평가·예측 모두 **train의 mean/std** 사용.
8. 다중분류 평가는 **confusion_matrix**(행=실제, 열=예측) + **classification_report**.
9. 우분투에선 `plt.savefig()`로 그래프 저장.
